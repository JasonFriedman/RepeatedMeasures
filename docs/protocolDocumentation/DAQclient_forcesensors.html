forcesensors

This client connects to a DAQ server to sample force sensors.

NameRequiredTypeDefault valueDescription
numchannelsYnumber[]Number of DAQ channels to sample
gainsYmatrix1Gains (one per channel), usually in N / V
offsetsYmatrix0Offsets (one per channel), usually in N
displayRangeXNn by 3 matrix[0,0,1]Range of values to show when using showPosition as the dot position on the x axis. The ith row corresponds to the ith sensor. Use the first two values to show the range of force values e.g. [0 10] to show values of 0 to 10 as the dot x position. Use the last value to show time, e.g. [0 0 1] will move across the screen in 1 s
displayRangeYNn by 3 matrix[0,1,0]Range of values to show when using showPosition as the dot position on the y axis. The ith row corresponds to the ith sensor. Use the first two values to show the range of force values e.g. [0 10] to show values of 0 to 10 as the dot y position. Use the last value to show time, e.g. [0 0 1] will move up the screen in 1 s.
offsetXNnumber0x offset when using showPosition
offsetYNnumber0y offset when using showPosition
showPositionColorNn by 3 matrix[255,0,0]color to show dot when using showPosition (each number between 0 and 255). The ith row corresponds to the ith sensor (if applicable)
filterTypeNstring[]Filter to apply if showing position (current only option is 'butter' for Butterworth filter). Note that filtering is only for onscreen display, and the saved data is not filtered. Use of a filter requires the signal processing toolkit. An empty string means no filtering.
filterOrderNnumber4If using a filter, which order to use
filterCutoffNmatrix8If using a filter, the cutoff frequency for a lowpass filter (in Hz). If 2 numbers are specified, a bandpass filter will be created.
samplerateNnumber150Sample rate (in Hz), only needed if using a filter
