colorInImage

The target(s) are defined by a color in an image

NameRequiredTypeDefault valueDescription
imageNumYesnumber1Image number to use (from those defined in setup)
startImageNumYesnumber1Image number for when the movement has started, for use with timing feedback
colorNo1 by 3 matrix[0,0,0]color in the image that signfies the target
colorStartNo1 by 3 matrix0color in the image that signfies the start point, for use with timing feedback
endtimeNonumber[0,0,0]The amount of time you need to be at the target for (in seconds)
pointsNon by 6 matrix[]Points to earn for this trial (based on position at the end of the trial). Should be a n*6 matrix. Each row includes (1) image number, (2-4) RGB color (5) number of points (6) tolerance of RGB color (city-block distance)
showPositionFeedbackNon by 2 matrix[]Show position feedback at certain frames in trial. Each row has 2 columns: (1) image number, (2) frame number. Can only use one of points or showPositionFeedback
showPathFeedbackNostring[]Show feedback on the path taken at the end of the trial. This should either be 1, or an image, or empty. If it is an image, the color of the image will specify the color of the feedback
showBackgroundNonumber[]Whether to show backgroud during the feedback presentation. Either an image number, or empty
distanceNonumber0allowable cityblock distance from the color
durationFeedbackNo1 by 2 matrix[]Whether to provide feedback about the duration. If so, it should be a 1x2 matrix (1st is minimum time, 2nd is maximum time)
