stateTransitions

Describes of how to transition from state to state. The current state corresponds to the image that will be shown (i.e. row from stimuli)

NameRequiredTypeDefault valueDescription
currentStateYesnumber1The state the program is currently in
positionYesmatrix1Either the position needed to trigger the change (index of the positions in targetPosition) if positionType=0, otherwise the color (1x3 matrix) from the image specified in positionType
distanceAllowedYesnumber1the maximum distance from the position that will trigger the change (i.e. distance must be less than this), ignored if positionType>0
minimumDistanceNonumber0the minimum distance from the position that will trigger the change (i.e. distance must be greater than this), ignored if positionType>0
penTouchingNonumber0whether the pen must be touching (If not using a tablet set to 0, for a tablet 0 = doesn't matter, 1 = must touch, 2 = must not touch)
timeElapsedNonumber0the minimum amount of time that has elasped since arriving in this state (in seconds)
newStateYesnumber1the state to move to if all the conditions are satisfied
positionTypeNonumber0positionType = 0 means to use positions from targetPosition, positionType>1 means to use a color from image number position (e.g. position=2 means use image 2). In this case specify the color in position
colorDistanceNonumber0allowable cityblock distance from the color (only used if positionType>0)
