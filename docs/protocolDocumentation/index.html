Protocol documentation

In the section experimentDescription.setup, there can be the fields below. There must be at least one of the clients.

NameRequiredTypeDefault valueDescription
imagesNcellarray[]Cell array of the filenames of the images (which are in the stimuli directory)
stringsNcellarray[]Strings to replace the default (for feedback, etc). Each item should have fields name (name of string to replace) and value (the new string to use)
soundsNcellarray[]Cell array of the filenames of .wav sound files to play (which are in the stimuli directory). All files need to have the same sample frequency and number of channels.
symbolsNcellarray[]A cell array of strings to display (e.g. {'<','>'})
monitorWidthNnumber70The width of the monitor in cm
viewingDistanceNnumber68The viewing distance in cm
xshiftNnumber0Horizontal shift of stimuli from the center of the screen
framerateNnumberNaNFrame rate to aim for (if the program can't keep up with the monitor frame rate
vrNstruct[]details of the vr setup (if vrstimulus is going to be used)
MCtriggerNnumber[]details of the DAQ card trigger (1 = input, 2 = output, empty if not used)
mouseTargetsNn by 4 matrix[]details of mouse targets (that can be clicked). Each row should be (x,y,width,height), in the range 0-1
boxesNn by 4 matrixNaNdetails of boxes to show on the screen. Each row should be (x,y,width,height), in the range 0-1
labelsNstruct[]list of labels (strings) that can be shown
DAQNstruct[]This client connects to a DAQ server to sample one or more channels (e.g. for a button box).
fastrakNstruct[]Connect to a fastrak server to record fastrak 6D data
gloveNstruct[]Connect to the cyberglove server to collect glove data
gloveemulatorNstruct[]Read glove data from a file. The files should be in the directory "emulated" and the name of the file to read is specified in each trial in the filename field.
glovetrackerNstruct[]This connects to both a glove and tracker (fastrak / liberty) server (or emulator).
keyboardNstruct[]Connect to a keyboard server to record keystroke information
libertyNstruct[]Connect to the liberty to sample from it
mouseNstruct[]Connect to the mouse server to sample the mouse location
optotrakNstruct[]Connects to the optotrak to sample 3D locations
tabletNstruct[]Connects to server to sample location and other information from a WACOM tablet
trackeremulatorNstruct[]Read tracker data from a file. The files should be in the directory "emulated" and the name of the file to read is specified in each trial in the filename field.
videoNstruct[]A client to connect to a program to record video


Each trial can have the following fields:

NameRequiredTypeDefault valueDescription
checkMovingNnumber0Whether to check the subject is still moving at this time
blockcounterNnumber1counter (used with triggers)
waitTimeBeforeNnumber0time to wait before showing the main stimulus (fixation is usually shown)
showPositionNnumber0whether the current location should be shown (depends on the input device)
drawFixationNnumber1whether to draw a fixation point during waitTimeBefore
targetNumNnumberNaNcorrect target number
textFeedbackNnumber1whether to display text feedback after the trial
textFeedbackShowBackgroundNnumber0whether to show the background when giving feedback
checkRecordingNnumber1whether to check the recording at the end of the trial (and give feedback if there is a problem, e.g. missing markers)
auditoryFeedbackNnumber1whether to provide auditory feedback (usually when incorrect)
filenameNstring[]filename to save the results
recordingTimeYnumber[]recording time in seconds
movementonsetNstruct[]how movement onset is determined
recordingNnumberNaNwhether to record
playAudioFileNstruct[]details of audio (wav) files to play
boxesNmatrix[]which boxes to show
labelsNmatrix[]which labels to show
beepNstruct[]beeps througout the trial
backgroundColorN1 by 3 matrix[0,0,0]Background color (1x3 RGB matrix, [0 0 0] = black, [255 255 255] = white)
