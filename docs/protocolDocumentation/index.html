Protocol documentation

In the section experimentDescription.setup, there can be the fields below.

NameRequiredTypeDefault valueDescription
imagesNocellarray[]Cell array of the filenames of the images (which are in the stimuli directory)
stringsNostruct[]Strings to replace the default (for feedback, etc).
soundsNocellarray[]Cell array of the filenames of .wav sound files to play (which are in the stimuli directory). All files need to have the same sample frequency and number of channels.
beepsZero or morestruct[]Details of custom beeps to play back during the trials
symbolsNocellarray[]A cell array of strings to display (e.g. {'<','>'})
staircasesZero or moreOne of:
QUEST
[]A cell array of staircases
monitorWidthNonumber70The width of the monitor in cm
monitorHeightNonumber30The height of the monitor in cm
viewingDistanceNonumber68The viewing distance in cm
xshiftNonumber0Horizontal shift of stimuli from the center of the screen
framerateNonumberNaNFrame rate to aim for (if the program can't keep up with the monitor frame rate)
vrNostruct[]details of the vr setup (if vrstimulus is going to be used)
MCtriggerNonumber[]details of the DAQ card trigger (1 = output, 2 = input, empty if not used)
incrementOnAbortNobooleanfalsewhether to move to the next trial when the trial is aborted (e.g. due to starting too early or late). If 0, then repeat the same trial, if 1 move to the next trial
mouseTargetsNon by 4 matrix[]details of mouse targets (that can be clicked). Each row should be (x,y,width,height), in the range 0-1
targetPositionNon by 2 matrix[]details of target positions. Each row should be (x,y) or (x,y,z), depending on the recording device used. These values can be overridden using the defineTarget response.
boxesNon by 4 matrixNaNdetails of boxes to show on the screen. Each row should be (x,y,width,height), in the range 0-1
labelsNostruct[]list of labels (strings) that can be shown
tactorsNostruct[]details of the tactor stimulation (vibration)
serialNostruct[]details of the serial port to be used for a trigger
parallelNostruct[]details of the parallel port to be used for a trigger
ATIforcesensorsNostruct[]Connects to the ATI force sensors server
DAQNostruct[]This client connects to a DAQ server to sample one or more channels (e.g. for a button box or force sensors).
arduinoNostruct[]This client connects to an arduino server to sample one or more digital channels
blueteraNostruct[]Connect to a bluetera server to record orientation + acceleration data
characterNostruct[]The is a client to connect to a character server (over TCP/IP)
fastrakNostruct[]Connect to a fastrak server to record fastrak 6D data
fixedgloveNostruct[]Use fixed joint angles in place of a glove.
fixedtrackerNostruct[]Use a fixed position / orientation in place of a tracker.
forcesensorsNostruct[]This client connects to a DAQ server to sample force sensors.
g4Nostruct[]Connect to the g4 to sample from it
gloveNostruct[]Connect to the cyberglove server to collect glove data
gloveemulatorNostruct[]Read glove data from a file. The files should be in the directory "emulated" and the name of the file to read is specified in each trial in the filename field.
glovetrackerNostruct[]This connects to both a glove and tracker (fastrak / liberty) server (or emulator).
goproNostruct[]A client to connect to a GoPro camera to record video
keyboardNostruct[]Connect to a keyboard server to record keystroke information
libertyNostruct[]Connect to the liberty to sample from it
mouseNostruct[]Connect to the mouse server to sample the mouse location
optotrakNostruct[]Connects to the optotrak to sample 3D locations
redamberNostruct[]Connect to a redamber server to record orientation + acceleration data
serialportNostruct[]Connect to a serial port and sample from it
tabletNostruct[]Connects to server to sample location and other information from a WACOM tablet
trackeremulatorNostruct[]Read tracker data from a file. The files should be in the directory "emulated" and the name of the file to read is specified in each trial in the filename field.
trakStarNostruct[]Connect to a trakStar server to record trakStar 6D data
videoNostruct[]A client to connect to a program to record video
